6. Контроль дедлайнов
Нужен автоматический планировщик задач.

Например:

если в дедлайн нет подтверждения → статус missed
сразу запускается автозамена

если исполнитель 3 раза сорвал срок → автопауза аккаунта
Это делается cron jobs / queue workers / scheduled tasks.

5. Какой user flow делать первым
Для исполнителя:
Flow 1 — вход
юзер жмет /start
бот показывает кнопку Открыть приложение
mini app получает Telegram user data
если нового юзера нет в БД — создаем
Flow 2 — онбординг
выбрать город
выбрать тематику
ввести username соцсети
указать средние просмотры
указать цену

Flow 3 — задания
список доступных заданий
открыть карточку
кнопка Принять

Flow 4 — выполнение

посмотреть ТЗ
кнопка Загрузить подтверждение
загрузить скрин
потом загрузить финальный скрин через 24 часа
Flow 5 — деньги
видеть баланс
история начислений
заявка на вывод
Это уже хватит на MVP.

8. Как именно “вайбкодить” через AI
Вот правильный подход:

Не просишь:

Просишь кусками.

 Создай Next.js Telegram Mini App с Tailwind и shadcn, который читает данные пользователя из Telegram WebApp SDK и показывает главную страницу с нижней навигацией: Главная, Задания, Мои, Профиль.

Потом:

 Сделай страницу заданий, подтягивай активные кампании из Supabase, отображай карточки с наградой, дедлайном и кнопкой Принять.
